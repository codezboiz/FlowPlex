Intro and tutorial.

PolyBrawl is a Smash Bros Style game, but it tries to combine retro features with more realistic modern features.
You have sent to the 2nd dimension due to a mass explosion of the Earth,
and have to fight to the death, because only one person can be sent back
to the 3rd dimension.





Controls

Player 1

Arrow keys to move

Slash for radar attack (if they are close, it hurts them)

Enter to heal


Player 2

WASD to move

Z for radar attack (if they are close, it hurts them)

E to heal

History Of PolyBrawl

PolyBrawl started development way before it was named PolyBrawl. One of our friends named Paxton suggested
Sylis to make a game. It was going to be called Collide and was planned to be a first person shooter and you were
able to go to different timelines. The project was ultimately scrapped for being to advanced for us back then.
Since I (Ryan) wanted to code in Ms. Barrier's class more, I decided to use an unblocked website
called PlayCanvas, which is a WebGL game engine. We created many new blank projects,
but most of them were scrapped.

On May 28, 2019, we created a project called Bloodshed-Battle. It was supposed to include animations,
but since none of us knew how to animate sprites in PlayCanvas that good,
we renamed it to PolyBrawl, so we could use simple sprites. The project's first beta
was published right before Summer Break of 2019.
Sylis went to Vermont, while I went to Bangladesh. During My time there, I
continued to update PolyBrawl,
while Sylis played Fortnite most of the Summer. It took about a month before the game
looked somewhat complete, but the UI controls weren't the best
(since I didn't really know how to use buttons at the time).
The gameplay was also still very basic, and collision had some bugs. (Those bugs were fixed in
the latest build, but new ones appeared later). The earliest build from that time that's still intact is called:


PolyBrawl BETA 1.6



I updated the game some more after that. When I came back, Sylis started helping on the game.
Together, we added more to polybrawl. The UI navigation still stuck around until January Of This Year. It was
updated to use buttons in the latest build. Although, this build from September
was when the game experimented with more new ideas. The September of 2019 build was called:

PolyBrawl BETA 1.9.3



We decided to re-create the UI Navigation, while creating the Christmas Update.
We would release the Christmas Update with the Old UI Navigation, and the New
UI Navigation with the New Years Update. The Christmas Update got corrupted,
because of the New UI Navigation in an alternate scene. We decided,
to cancel the Christmas Update, and we released the New Years Update
exactly at midnight right when the new year started. This build
is also one of the most recent, only being made a few months ago. The New Years Update was called:

PolyBrawl BUILD 2.0 (2020)
